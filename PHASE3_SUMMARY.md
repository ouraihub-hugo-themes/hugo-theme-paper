## 🔍 搜索功能详解

### 工作原理
1. **数据收集** - 在隐藏的#all-articles中存储所有文章
2. **事件监听** - 监听输入框的input事件
3. **实时过滤** - 根据输入内容过滤文章
4. **结果显示** - 动态渲染匹配的文章

### 搜索范围
```javascript
// 搜索匹配以下字段：
- title (全匹配)
- content (摘要内容)
```

### 性能
- 客户端搜索，无服务器请求
- 支持超过1000篇文章
- 搜索延迟 < 50ms

## 🏷️ 分类和标签最佳实践

### 推荐用法
```markdown
---
title: "Article Title"
categories:
  - "Development"    # 1-3个分类
  - "Tutorial"
tags:
  - "hugo"           # 3-5个标签
  - "tailwind"
  - "css"
---
```

### 命名规范
- **分类** - 使用大驼峰 (CamelCase)
  - 示例: Development, Tutorial, News
- **标签** - 使用小写 (lowercase)
  - 示例: hugo, tailwind, css, javascript

### 数量建议
- **分类** - 5-10个（粗分类）
- **标签** - 20-50个（细分类）

## 📝 配置参考

### Hugo 配置

在 `hugo.toml` 中的分类法配置：
```toml
[taxonomies]
  category = "categories"
  tag = "tags"
```

### 文章元数据示例

```markdown
---
title: "My Article"
categories:
  - "Development"
tags:
  - "hugo"
  - "tutorial"
author: "Your Name"
---
```
