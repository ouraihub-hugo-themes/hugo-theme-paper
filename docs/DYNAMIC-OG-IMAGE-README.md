## 💡 核心概念

### 什么是动态 OG 图片？

当你分享文章到社交媒体时，**自动**根据文章内容（关键词、标签等）从 Unsplash 获取相关图片。

### 为什么需要？

**传统方式:**
```
写文章 → 设计图片 → 导出图片 → 上传图片 → 在 frontmatter 指定
```
**耗时**: 每篇文章 10-30 分钟

**动态方式:**
```
写文章 → 添加关键词 → 完成！
```
**耗时**: 0 分钟（自动完成）

### 效果对比

**没有动态 OG 图片:**
- 所有文章用同一张默认图
- 不吸引人
- 点击率低

**有动态 OG 图片:**
- 每篇文章有相关的精美图片
- 更吸引眼球
- 点击率提升 30-50%

## 🎨 使用示例

### 最简单的用法

**配置一次:**
```toml
[ogImage]
  mode = "unsplash"
```

**每篇文章:**
```yaml
---
title: "我的文章"
keywords:
  - hugo
  - blog
---
```

**自动生成:**
```
https://source.unsplash.com/1200x630/?hugo,blog
```

**就这么简单！**

## ❓ 常见问题

### Q: 需要 API key 吗？
A: 不需要！Unsplash Source API 完全免费，无需注册。

### Q: 有请求限制吗？
A: 每小时 50 次。对于静态网站足够（图片 URL 在构建时生成）。

### Q: 图片质量如何？
A: Unsplash 的图片都是专业摄影师拍摄，质量很高。

### Q: 可以用自己的图片吗？
A: 可以！在 frontmatter 中指定 `cover` 或 `image` 即可。

### Q: 如果关键词找不到图片怎么办？
A: 会使用随机图片，或者使用配置的 fallback 图片。

### Q: 可以应用到其他 Hugo 主题吗？
A: 可以！代码是通用的，只需复制 partial 模板和配置即可。

### Q: 会影响网站性能吗？
A: 不会！图片由 Unsplash CDN 提供，不占用你的服务器资源。

### Q: 可以离线使用吗？
A: 图片 URL 在构建时生成，但图片本身需要在线加载。可以配置 fallback 本地图片。
